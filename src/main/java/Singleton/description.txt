1. Singleton Pattern (Паттерн "Одиночка"):
Вы можете использовать Singleton для создания центрального модуля,
который управляет доступом к базе данных и хранит информацию о рейсах, билетах и пользователях.
Это обеспечит единство доступа к данным и управление ими.

можно запихать когда админ создает рейсы
